private void ChangeBoxIdOnly(BoundingBox box, int newId)
{
    if (box == null) return;
    int oldId = GetBoxId(box);
    if (oldId == newId) return;

    var origRect = box.Rectangle;
    var origLabel = box.Label;

    SetBoxId(box, box.Label, newId);   // ✓ 이 박스만

    AddUndoAction(new UndoAction {
        Type = UndoActionType.ModifyBox,
        Box = CloneBoundingBox(box),    // 변경 후 상태 (Undo 핸들러 lookup 대상)
        OriginalRectangle = origRect,
        OriginalLabel = origLabel,
        OriginalObjectId = oldId,       // 변경 전 값 (복원용)
    });

    // Waypoint.ObjectId 자동 변경 없음 — D-02 locked
    UpdateObjectInfo(box); UpdateBboxListDisplay(); pictureBoxVideo.Invalidate();
}

### NEW-01 (ID 승계)

**Before** — `currentAssignedId` 만 사용, Entry 에 기존 박스 있어도 조회하지 않음:
```csharp
drawingBox = new BoundingBox
{
    ...
    PersonId = currentSelectedLabel == "person" ? currentAssignedId : 0,   // ❌ Entry 무시
    VehicleId = currentSelectedLabel == "vehicle" ? currentAssignedId : 0,
    EventId = currentSelectedLabel == "event" ? currentAssignedId : 0,

```

**After** — 5단계 ID 승계:
```csharp
int resolvedId = ResolveIdForNewBox(currentSelectedLabel, currentFrameIndex);

drawingBox = new BoundingBox
{
    ...
    PersonId = currentSelectedLabel == "person" ? resolvedId : 0,
    VehicleId = currentSelectedLabel == "vehicle" ? resolvedId : 0,
    EventId = currentSelectedLabel == "event" ? resolvedId : 0,
};

private int ResolveIdForNewBox(string label, int currentFrameIndex)
{
    // (1) Waypoint 포함 → Entry 박스 ID
    var containingWaypoint = waypointMarkers.FirstOrDefault(w =>
        w.Label == label
        && currentFrameIndex >= w.EntryFrame && currentFrameIndex <= w.ExitFrame);
    if (containingWaypoint != null)
    {
        var entryBox = GetBboxesForFrame(containingWaypoint.EntryFrame)
            .FirstOrDefault(b => b.Label == label && !b.IsDeleted);
        if (entryBox != null) return GetBoxId(entryBox);
    }

    // (2) entryFrameIndex 있고 현재가 Entry 이후
    if (entryFrameIndex.HasValue && currentFrameIndex >= entryFrameIndex.Value)
    {
        var entryBox = GetBboxesForFrame(entryFrameIndex.Value)
            .FirstOrDefault(b => b.Label == label && !b.IsDeleted);
        if (entryBox != null) return GetBoxId(entryBox);
    }

    // (3) 같은 프레임 selectedBox
    if (selectedBox != null && selectedBox.Label == label && selectedBox.FrameIndex == currentFrameIndex)
        return GetBoxId(selectedBox);

    // (4) Ctrl+숫자 지정값
    if (currentAssignedId >= 1) return currentAssignedId;

    // (5) 기본
    return 1;
}
```

## Decisions Made

- **ChangeBoxIdOnly Undo 계약 정정 (Rule 1)**: 플랜 원안은 `AddUndoAction` 을 `SetBoxId` 이전에 호출하며 Original* 필드를 미세팅했음. advisor 지적대로 Undo 핸들러(line 2270-2271) 의 `GetBoxId(b) == GetBoxId(action.Box)` lookup 은 변경 후 ID 로 수행되므로 `action.Box` 는 SetBoxId **후** clone 해야 하며, 복원값은 Original* 로 전달해야 함. 교정된 코드는 핸들러 계약을 완전히 준수.
- **Alt+숫자 포커스 가드 예외 추가 (Rule 2)**: 플랜은 Ctrl+숫자/Ctrl+N 만 예외 명시했으나 Alt+숫자(Person 11-20) 경로도 같은 ActiveControl 가드에 의해 차단되고 있었음. 동일 결함이므로 `isIdShortcut` 조건에 `e.Alt && IsIdAssignmentKey` 를 추가.
- **Waypoint 포함 검사 우선순위를 entryFrameIndex 위에 배치 (ResolveIdForNewBox)**: 사용자가 Entry 마커를 해제(Escape) 후 기존 Waypoint 구간 내부로 이동하여 새 박스를 그리는 시나리오에서도 해당 Waypoint 의 Entry ID 가 승계되도록.
- **Waypoint.ObjectId 자동 변경 금지 (D-02 locked)**: ChangeBoxIdOnly 는 Waypoint.ObjectId 를 건드리지 않음. Waypoint 단위 변경은 의도적으로 삭제-재생성 UX 강제 — ID 관리 일관성을 보장.
- **Propagation 경로 3곳(line 2289/2313/2346) 미수정**: 이미 source box 의 EventId/PersonId/VehicleId 를 직접 파생받으므로 NEW-01 범위 외.
- **Ctrl+N 'all boxes' 시 같은 Label 첫 Entry 박스 ID 적용**: 다객체 Entry 에서는 ID 수렴 가능 — 정밀 매칭은 selectedBox 를 지정한 뒤 Ctrl+N 을 권장.
