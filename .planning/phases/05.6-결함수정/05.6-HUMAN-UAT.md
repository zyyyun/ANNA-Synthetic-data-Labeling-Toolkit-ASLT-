**Investigation summary (2026-04-24):**

코드 감사 결과, 보고 증상("Ctrl+{digit} 후 Rectangle 이 처음 설정한 박스 좌표로 고정")을 일으키는 경로를 **발견하지 못했습니다**. 감사 범위:

- `ChangeBoxIdOnly(box, newId)` (MainForm.cs:2890-2917) — `box.Rectangle` 을 쓰지 않음 (PersonId/VehicleId/EventId 만 변경)
- `SetBoxId` (MainForm.cs:3084-3089) — 3 개 ID 필드만
- `CloneBoundingBox` — deep-copy, 원본 불변
- `AddUndoAction` — `_isDirty=true` + stack push, 액션 미적용
- `_bboxByFrame` 캐시 — FrameIndex 만 키, live 참조 (stale 없음)
- `UpdateBboxListDisplay` → `UpdateBboxPanel` → `Controls.Clear()` — UI 컨트롤 dispose, BBOX 데이터 미변경
- `Propagate*BoxFromCurrentFrame` — drag/resize MouseUp 에서만 호출
- 모든 `box.Rectangle = ` 쓰기 지점 13곳 확인 — ID 변경 경로 연관 0건

세부는 `05.6-CONTEXT.md` D-21 참조.

**Action taken:**

1. **진단 계측 추가** (MainForm.cs:2890-2930): `ChangeBoxIdOnly` 의 entry/exit 에 `[NEW-08]` prefix Debug.WriteLine 추가. box hash + rect before/after 기록. Visual Studio Debug Output 또는 DebugView (Sysinternals) 에서 확인 가능.
2. **CONTEXT.md D-21** 에 진단 이력 + 재현 discriminating questions 등록.
3. **Speculative fix 금지** — 재현 불가 상태에서 cargo-cult 방어 코드 추가 시 grep 검증만 통과하고 실결함은 미수정될 위험. GS 인증 감사관 노이즈만 증가.
