## JsonService.DeleteJsonForVideo 시그니처 + 감사 로그

```csharp
/// <summary>
/// DF-1-04 (D-08) / DF-1-05 (D-09): 비디오에 대응하는 COCO JSON 파일을 삭제하고
/// 감사 이벤트를 기록한다. 경로 트래버설 방지 검증 포함.
/// Wave 5 에서 LogService.AuditJsonDelete 래퍼로 전환될 수 있음.
/// </summary>
public bool DeleteJsonForVideo(string videoFilePath)
{
    string? jsonPath = ResolveJsonPath(videoFilePath);
    if (string.IsNullOrEmpty(jsonPath) || !File.Exists(jsonPath))
        return false;

    string? videoDir = Path.GetDirectoryName(videoFilePath);
    if (!string.IsNullOrEmpty(videoDir) && !PathValidator.IsPathSafe(jsonPath, videoDir))
    {
        Log.Warning("[보안] 삭제 경로 트래버설 감지: {Path}", jsonPath);
        return false;
    }

    try
    {
        File.Delete(jsonPath);
        Log.Information("[AUDIT] JSON 파일 삭제: {Path}", jsonPath);   // ← Wave 5 치환 대상
        return true;
    }
    catch (Exception ex)
    {
        Log.Error(ex, "[JSON 삭제 오류] {Path}", jsonPath);
        throw;
    }
}
```

**감사 로그 라인 (현재 포맷)**: `[AUDIT] JSON 파일 삭제: {Path}`
**Wave 5 에서 예상 치환**: `LogService.AuditJsonDelete(jsonPath)` (새 LogService 메서드 정의 후 교체)

## Decisions Made

- **D-07/D-08 canonical 문자열 정확 사용**: `"Waypoint 구간 BBOX가 전부 삭제됩니다. 해당 Waypoint도 함께 삭제하시겠습니까?"` + `"Waypoint가 없어 해당 JSON 파일은 삭제됩니다. 삭제하시겠습니까?"` — byte-for-byte grep 매치 확인.
- **_autoSavedJsonPath 단일 choke-point (advisor 가이드)**: `SaveCurrentLabelingData` 내부에서 한 번만 세팅하여 6개 caller(btnSelectFolder_Click, Task.Run 경로, Waypoint-트리거 자동 저장 3곳, FormClosing 저장 경로) 전체 커버. Caller 별 중복 세팅 회피로 유지보수성 확보.
- **btnDeleteLabel_Click 도 Prompt 호출 (advisor 가이드)**: 사이드바 삭제 버튼은 Delete/G 키와 동일한 사용자 주도 경로 — 동등 대우.
- **OnFormClosing 롤백 삭제 확장 (Rule 2 — plan step 7 scope 확장)**: 원 plan 에 "FormClosing 에 No 분기 있으면 동일 로직 적용" 이라는 조건부 지침이 있었으나 현재 코드에는 Yes/Cancel 만 명시 분기되어 있고 No 는 암묵적 pass-through 였음. DF-1-05 의 사용자 기대 UX(미저장 편집을 저장 안 하겠다 → JSON 잔존 금지)는 앱 종료에서도 동일하므로 No 분기를 명시 구현하고 롤백 삭제 적용.
- **DeleteJsonForVideo 에 PathValidator 방어선 추가 (Rule 2 — plan 원안 누락분 보강)**: 기존 DeleteJsonFileForVideo 는 PathValidator.IsPathSafe 를 수행하므로 신규 메서드도 동일 가드 필요. SECU-04 일관성.
- **Legacy DeleteJsonFileForVideo → DeleteJsonForVideo 위임 (Rule 1 — code smell 해소)**: 두 메서드가 거의 동일 동작을 하는 중복 경로는 유지보수 비용. Hint 경로 있으면 즉시 감사 로그 후 삭제, 없으면 새 메서드로 위임. `btnDeleteJson_Click` 도 이제 [AUDIT] 로그를 기록하는 경로를 타게 됨.
