## DF-1-13 Before/After

### Before (크래시 재현 — PDF p.7 스크린샷)

```
조건: 앱 실행 직후, 영상 로드 없이 Entry 버튼 클릭
→ btnEntry_Click → SetEntryMarker() 진입
→ _videoService.CurrentFrameIndex = 0, _videoService.Fps = 0 (double)
→ 0 / 0 = NaN
→ TimeSpan.FromSeconds(NaN) throw ArgumentException (".NET: TimeSpan doesn't support NaN")
→ unhandled exception → 앱 종료 또는 WinForms 예외 다이얼로그
```

### After (가드 적용)

```
조건: 동일
→ btnEntry_Click 진입
→ selectedWaypoint null 경로 진입
→ if (!_videoService.IsVideoLoaded) → true
→ MessageBox.Show("영상을 먼저 로드해 주십시오.", "알림", OK, Information)
→ return (TimeSpan 호출 자체에 도달하지 않음)
→ 앱 정상 동작 유지
```

`btnExit_Click`, `Keys.E`, `Keys.X` 단축키 모두 동일 패턴. Keys.E/X는 이미 line 2446의 기존 early-return으로 보호되므로 추가 수정 없이 주석만 고정. 추가로 `SetEntryMarker` / `SetExitMarkerAndCreateWaypoint` 자체에도 이중 방어선 배치.

## Decisions Made

- **가드 배치 위치**: public 진입점(버튼 핸들러)과 private helper(SetEntryMarker/SetExitMarkerAndCreateWaypoint) 양쪽에 배치 — 향후 리팩토링/신규 호출자 추가 시에도 안전. private helper의 early-return은 MessageBox 없이 조용히 return(이미 public에서 메시지 표시됨).
- **selectedWaypoint != null 분기도 가드**: 기존 USAB-09 기능(Waypoint 선택 시 Entry/Exit가 해당 프레임으로 이동)도 영상이 로드되지 않으면 의미 없으므로 LoadFrame 호출 전 동일 메시지 표시.
- **MainForm_KeyDown line 2446 기존 가드 유지**: 이미 Keys.E/X 브랜치 앞에 있음. 삭제 방지용 주석만 추가 — 추가 코드 없음.
- **MessageBoxIcon.Information 선정**: Waypoint 생성 실패 등 기존 경고와 구분. 사용자 실수 (로드 순서 잘못) 안내는 Information이 ISO 사용성 가이드에 부합.
