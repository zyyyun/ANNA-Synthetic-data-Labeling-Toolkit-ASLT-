## ❓ 사용자 확인 필요 사항

1. **Ctrl+N 정체 확정**:
   - 오류 메시지(`MainForm.cs:996`)에만 언급되고 **실제 구현 없음**.
   - QA가 "Ctrl+N — 여기서 자동으로 객체 ID가 맞춰지면 해결됨(편의성 ↑)"이라고 제안.
   - → 이번 플랜에서 **Ctrl+N을 "Exit 프레임 BBOX의 ID를 Entry와 자동 일치" 기능으로 신규 구현**할까요, 아니면 메시지에서 "Ctrl+N" 문구만 **제거**(= Ctrl+1~0 매뉴얼 수정만 안내)할까요?

2. **DF-1-17 보안 강화 범위**:
   - (a) 현 상태 인정 (보존 기간 30일만 명시 강화, 보안 요건 미포함) — KTC가 재이의 가능성 있음
   - (b) 감사 이벤트 커버리지 확장만 추가 (영상 로드/BBOX/Export 등)
   - (c) (b) + HMAC 기반 무결성 검증 로그 체인
   - (d) (c) + AES 암호화
   - → 어느 수준까지 갈까요? (GS인증 1등급 기준으로는 최소 (b)+무결성 권장)

3. **DF-1-17 보존 기간**:
   - 현재 30일. "요구되는 기간"이 계약/시방서상 명시된 값이 있나요? (예: 1년, 3년)

4. **NEW-05 해석**: "해당 클래스 전체 객체가 같이 변경됨" — 이게 **코드 상 의도된 동작**(`ChangeBoxIdWithinWaypoint`이 같은 oldId 전부 변경)일 수도 있습니다. "Waypoint 구간에서 같은 ID 가진 박스들을 하나의 객체로 보고 동시 변경"이 **원래 설계**였는지, **버그**로 봐야 하는지 확인 필요.

5. **QA 텍스트 파일 "[수동 추적 관련]" 섹션**: 내용 비어있음(line 61-64). 누락인가요, 아니면 다음 배치에서 채워지나요?
