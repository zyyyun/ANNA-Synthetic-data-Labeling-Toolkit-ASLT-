효과: WM_TIMER fire 빈도가 짧아져 OS scheduler 의 강제 반올림 (33→45ms) 영향을 받지 않음. OnTimerTick 내부 `elapsedMs >= msPerFrame` 체크가 33.3ms 마다 advance 결정 (8ms 입도로 polling). 한 tick 당 처리 비용은 framesToMove=0 인 경우 elapsedMs 계산만이라 무시 가능.

**변경 2 — lastFrameTime drift 차단 (timerPlayback_Tick 안, 현재 line 1080):**

변경 전:
```csharp
            if (framesToMove > 0)
            {
                int nextFrame = Math.Min(_videoService.TotalFrames - 1, _videoService.CurrentFrameIndex + framesToMove);
                LoadFrame(nextFrame);
                lastFrameTime = currentTime;
```

변경 후:
```csharp
            if (framesToMove > 0)
            {
                int nextFrame = Math.Min(_videoService.TotalFrames - 1, _videoService.CurrentFrameIndex + framesToMove);
                LoadFrame(nextFrame);
                lastFrameTime += (long)(framesToMove * msPerFrame);
```

효과: 이상적 frame 시간을 누적. Timer 가 47ms 후 fire 해도 다음 tick 에서 누적된 14ms 가 합산되어 framesToMove=2 (catch-up) 발생 가능. drift 영원히 차단.
